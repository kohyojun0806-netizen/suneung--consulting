이번 세션의 최우선 목표는 UI 디자인 개선이며, 3AGENT + GSD 체계를 반드시 유지한다.

[반드시 지킬 프로세스]
1) PLANER: 이번 스프린트 범위/완료기준/평가기준을 먼저 정의
2) Generator: 구현
3) evaluator: 검증 + 점수 + 피드백
4) evaluator 만족 전까지 반복

[strict evaluator 규칙]
- 최소 5회 반복
- target score 95
- Playwright PASS 필수
- sandbox fallback pass 금지
- 검증 명령: verify:ingest + verify:catalog + build + e2e

[이번 세션 핵심 방향]
- UI를 난잡하지 않고 유기적으로 개선
- 상위권 대상 교재/N제/실모 데이터 품질 강화
- sourceRefs가 있는 근거 기반 추천 유지
- 문서 기록(Contract/Log/Audit) 누락 금지

[작업 시작 시 출력 형식]
1) 현재 상태 8~12줄 요약
2) 이번 스프린트 계약(PLANER 산출물)
3) 구현 계획(Generator)
4) 검증 계획(evaluator)

[실행 명령]
- npm run gsd:sprint-loop -- --project ui-data-priority-20260330 --min-iterations 5 --max-iterations 7 --target-score 95 --strict-evaluator true --with-e2e true --sandbox-fallback false
- npm run gsd:verify-work -- --project ui-data-priority-20260330 --strict-evaluator true --with-e2e true --sandbox-fallback false

- 변경 파일 목록
