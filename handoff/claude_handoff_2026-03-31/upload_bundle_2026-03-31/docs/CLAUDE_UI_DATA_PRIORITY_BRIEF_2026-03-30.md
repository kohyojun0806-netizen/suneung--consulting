# Claude Priority Brief (UI First + Evidence Expansion)

## 최우선 과제
- 1순위: UI 디자인 전면 개선
- 현재 UI는 품질/독창성이 낮으므로, 안전한 개선이 아니라 명확한 컨셉 기반 리디자인 필요
- 스프린트를 다회 반복하며 evaluator 피드백을 강하게 반영

## 필수 개발 구조
- PLANER: 무엇을 만들지 정의(문제, 범위, 완료기준)
- Generator: 어떻게 만들지 구현(디자인 품질/독창성 가중치 최우선)
- evaluator: 점수화 + 피드백 + 재평가
- 기본 루프: 최소 5회 반복 + 최적 근접까지 추가 반복

## 데이터 방향 (UI 다음 우선순위)
1. 실제 학생 성공 사례 데이터 확장
- 커뮤니티/강사 게시판의 후기/칼럼/댓글 기반 사례 수집
- 신뢰도 기준(출처, 구체성, 재현 가능성, 기간/점수 변화 명시)으로 선별

2. 학생 질문 신호 데이터 확장
- 유튜브 댓글 데이터 기반 질문 패턴 수집
- 특히 `악어오름` 관련 영상/댓글 축을 우선 반영

3. 강사/교재 추천 데이터 확장
- 현재 추천 데이터량이 부족하므로 강사/교재 데이터 대폭 확대
- 상위권(1~3등급) 대상:
- 시대인재 단과, 대치 두각 단과 등에서
- “누구를 듣는다”를 넘어서 “언제(시기) 어떤 콘텐츠/수업을 듣는지”까지 명확히 제시

## 산출물 요구
- UI 스프린트별 계약서/평가표/피드백 로그
- 데이터 수집 로그(출처, 수집 일시, 필터링 기준, 반영/보류 사유)
- 최종 추천 로직 변경 내역
- 검증 결과(`build`, 가능한 경우 `e2e`)

## 품질 기준
- 디자인: 독창성, 일관성, 모바일 완성도, 가독성
- 데이터: 신뢰도, 최신성, 재현성, 추천 근거 명확성
- 시스템: 회귀 없음, API 응답 안정성, 문서화 완결성
